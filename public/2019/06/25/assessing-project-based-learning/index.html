Assessing Project Based Learning

2019-06-25

Summary of the Student Project

A “customer” is requesting a blog assessing the state of Kathmandu air pollution. Students teams will design, create, and publish a website based on research and original data analysis for a customer by integrating 21st century technology development skills.

During this project students will demonstrate mastery of the following competencies1 [rubric]:

  1. Presentation (of website)
    • Motivation for website
    • Design and building experience
    • Discuss features
    • Discuss feedback received/given
    • Used Rmarkdown-based presentation hosted on GitHub Pages
  2. Revisions (via GitHub)
    • Teams assess each other by providing student feedback through GitHub’s social features2
    • Edit/proofread others’ work commenting and providing constructive suggestions for changes
    • Teams incorporate changes as desired and can justify why they were or were not incorporated
  3. Publishing (to GitHub)
    • Register a new domain for website (setup Netlify w/ GitHub)
    • Build and deploy (when changes to website are made)
  4. Creating (a website)
    • Create original data analysis using tidyverse package
    • Create a new project in RStudio using blogdown package
    • Incorporates (and improves upon as needed) previously completed and submitted coursework
  5. Designing (a website)
    • Create an effective project outline.
    • Divides responsibilities amongst team:
      • Project manager
      • Marketing
      • User experience
      • Technical
  6. Programming (can use the Rmarkdown and R languages appropriately and effectively)
    • Basic formatting techniques (e.g., bold/italics, headings, footnotes)
    • Code blocks to show R code
    • Citations

Student Monitoring and Feedback

Since GitHub is a service that is open to the public by default, this will allow access to both students and teachers to monitor and provide feedback. This is also a unique opportunity for students to learn how to conduct themselves in a public forum.

Teams will suggest changes to other teams’ projects via GitHub “pull requests”.

Pull Requests

The teams will communicate and provide feedback with each other using GitHub’s advanced commenting features such as inline commenting.

Inline Commenting

Teams will be expected to consider each others’ feedback, responding back to all suggestions exlpaining why teams agree or disagree with suggestions.

Giving/Receiving Feedback

Using GitHub as the platform to publish student projects, several things are accomplished.

For Students

  • Authentic learning is promoted by using a real-world design environment.3
  • Students showcase a self-inspiring portfolio of work!4
  • Students learn to give and receive feedback.
  • Students develop meta-cognitive skills through feedback sessions.

For Teachers

  • Can freely monitor and provide direct feedback to students.
  • Student work can be directly assessed.
